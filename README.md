Постановка задачи:
Разработка сетевого приложения. Клиент на базе протокола TFTP. Реализовать передачу файлов, реализация пяти элементов сообщений (запрос на чтение, запрос на запись, данные, подтверждение, ошибка). Cервер использовать стандартный.

Реализация
Разработанное приложение представляет собой консольного TFTP-клиента для опе-рационной системы Linux. Программа написана на языке C с использованием стандартных сокетов Беркли. Клиент взаимодействует со стандартным TFTP-сервером по протоколу UDP и поддерживает два режима работы: GET (скачивание файла с сервера) и PUT (загрузка файла на сервер).
В соответствии с заданием в программе реализованы все пять типов сообщений протокола TFTP. Первый тип — RRQ (запрос на чтение) с кодом операции 1. Второй тип — WRQ (запрос на запись) с кодом операции 2. Оба формируются функцией send_request(), которая записывает в буфер код операции, имя файла с завершающим нулевым байтом и режим передачи "octet" с завершающим нулевым байтом. Третий тип — DATA (пакет дан-ных) с кодом операции 3, формируется функцией send_data(). Пакет состоит из четырёх-байтового заголовка и блока данных размером до 512 байт. Четвёртый тип — ACK (под-тверждение) с кодом операции 4, формируется функцией send_ack(). Пакет имеет фикси-рованный размер четыре байта и содержит номер подтверждаемого блока. Пятый тип — ERROR (ошибка) с кодом операции 5, формируется функцией send_error(). Пакет содержит код ошибки и текстовое сообщение.

Запуск клиента осуществляется командой:
./tftp_client <IP-адрес сервера> <get|put> <имя_файла>
